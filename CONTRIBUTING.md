# Contributing to QuicDraw

Thank you for your interest in contributing to QuicDraw! We are always delighted to welcome new contributors!
For general contributions and community guidelines, please see the [cyberark community documentation](https://github.com/cyberark/community/).

## Found an Issue?

If you have found a bug, please raise an issue on the QuicDraw repo: https://github.com/cyberark/quicdraw/issues

## Found a Vulnerability?

If you think you have found a vulnerability in QuicDraw, please refer to [Security](SECURITY.md)

We are always very grateful to researchers who report vulnerabilities responsibly.

## Development

We recommend using Python +3.10

### General Steps for Contributing (Creating a Pull Request)

1. Fork the project.
2. Clone your fork.

```
# Clone the repository (fork)
git clone https://github.com/repo/fork
```

3. Install the project's requirements

```
# Install the project's requirements:
pip install -r requirements.txt
```

4. Make local changes to your fork by editing files
5. Commit your changes.
6. Push your local changes to the remote server.
7. Create a new Pull Request.

From here, your pull request will be reviewed, and once it is merged into the project. Congratulations, you're a contributor!
